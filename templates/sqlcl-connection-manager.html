003 — SQLcl Connection Manager

A technical deep dive into SQLcl Connection Manager (CM) for Oracle 26ai developers and DBAs. If your day spans multiple environments — a local containerised 26ai instance, a cloud-hosted PDB on OCI, an Autonomous Database accessed via wallet, and a vector store for AI pipelines — Connection Manager eliminates the tedium of managing all those connection strings manually.

SQLcl CM acts as a named connection registry that persists across sessions, integrates with the -name flag at startup, and satisfies the -savepwd requirement for the SQLcl MCP server — making it a hard dependency for any Oracle 26ai AI Optimizer, RAG pipeline, or MCP-based automation workflow.

The SQLcl MCP server requires a saved connection with stored password (-savepwd) to operate without interactive prompts. Properly configured Connection Manager entries are a prerequisite for any Oracle 26ai AI or automation workflow.

Core Commands

CommandDescription
conn -save <name> -savepwd <connstr>Save a named connection with stored password
conn -save <name> -replace -savepwd <connstr>Update an existing named connection
sql -name <name>Connect using a saved connection name
cm listList all saved connections
cm show <name>Show connection details
cm test <name>Test connectivity
cm delete -conn <name>Remove a connection
cm export > file.jsonExport connection names to JSON
cm import file.jsonImport connections from JSON

Saving Connections

Local PDB

conn -save dev26ai -savepwd \
  system/<your_password>@localhost:1521/freepdb1

cm test dev26ai

Autonomous Database (Wallet)

export TNS_ADMIN=/home/oracle/wallets/adb_dev
conn -save adb_dev -savepwd \
  admin/<your_password>@myatp_high

cm test adb_dev

Vector Store (Oracle 26ai)

conn -save vs_dev -savepwd \
  vector_admin/<your_password>@localhost:1521/freepdb1

conn -save ai_opt -savepwd \
  ai_opt_user/<your_password>@localhost:1521/freepdb1

OKE / Kubernetes NodePort

conn -save oke_prod -savepwd \
  system/<your_password>@127.0.0.1:1521/FREEPDB1

MCP Server Integration — Claude Desktop

Connection Manager is architecturally essential for Oracle 26ai AI workflows. The SQLcl MCP server launches SQLcl as a subprocess with no interactive terminal — the -name flag with a -savepwd connection is the only supported authentication path.

// claude_desktop_config.json
{
  "mcpServers": {
    "oracle-dev": {
      "command": "/opt/sqlcl/bin/sql",
      "args": ["-name", "dev26ai"]
    },
    "oracle-adb": {
      "command": "/opt/sqlcl/bin/sql",
      "args": ["-name", "adb_dev"]
    },
    "oracle-vectorstore": {
      "command": "/opt/sqlcl/bin/sql",
      "args": ["-name", "vs_dev"]
    }
  }
}
Never put credentials in claude_desktop_config.json or MCP server arguments. Using -name keeps credentials out of process argument lists and version-controlled config files.

CI/CD Automation

For headless environments and GitHub Actions pipelines, connection entries can be bootstrapped without manual interaction:

#!/bin/bash
# setup_connections.sh
SQLCL=/opt/sqlcl/bin/sql

$SQLCL -S /nolog << EOF
conn -save dev26ai -replace -savepwd system/${DB_PASSWORD}@localhost:1521/freepdb1
cm test dev26ai
EXIT
EOF
