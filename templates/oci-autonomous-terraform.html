002 — Oracle ADB + ORM Stack

A complete Infrastructure-as-Code solution for deploying Oracle Autonomous Database on Oracle Cloud Infrastructure (OCI) using Resource Manager Stack (ORM) and Terraform. This project automates the provisioning of an enterprise-grade Oracle ADB instance, a compute instance, and all required networking — from zero to working database in under 5 minutes.

The stack is designed for Oracle 26ai developers who need a repeatable, version-controlled way to spin up ADB environments for development, testing, or production. It uses OCI Resource Manager so no local Terraform installation is needed.

Full source code and documentation available on GitHub: github.com/ViliTajnic/OCI-Autonomous-Terraform-IaC

What Gets Deployed

Oracle Autonomous Database

  • Always Free tier — 20GB storage, 1 OCPU
  • OLTP workload type, auto-scaling disabled
  • Automatic backups enabled
  • Self-managing: patches, tunes, and scales itself

Compute Instance

  • VM.Standard.E2.1.Micro (Always Free)
  • Oracle Linux 8, Python 3.x pre-installed
  • SSH access configured via public key

Network Infrastructure

  • VCN with 10.0.0.0/16 CIDR
  • Public subnet (10.0.1.0/24)
  • Internet Gateway and route tables
  • Security groups for SSH (port 22)

Stack Outputs

  • Instance public IP address
  • SSH connection command
  • ADB service console URL

Terraform Structure

The entire infrastructure is defined in 4 Terraform files:

OCI-Autonomous-Terraform-IaC/
├── main.tf          # Core infrastructure (ADB, compute, networking)
├── variables.tf     # Input parameters (password, SSH key, compartment)
├── outputs.tf       # Stack outputs (IP, SSH command, console URL)
├── schema.yaml      # ORM UI configuration
└── scripts/
    ├── requirements.txt
    ├── quick_test.py
    ├── test_basic_connection.py
    ├── sample_queries.py
    └── troubleshoot_connection.py

Deployment Steps

  1. Clone or download the repository and create a ZIP of the Terraform files
  2. Go to OCI Console → Developer Services → Resource Manager → Create Stack
  3. Upload the ZIP file, fill in variables (password, SSH key, compartment OCID)
  4. Click Plan, review, then Apply — infrastructure is ready in minutes
  5. Download the ADB wallet from OCI Console, upload to compute instance
  6. Run python3 quick_test.py to verify connectivity

Python Connection Scripts

Five ready-to-use Python scripts are included for testing and exploring the ADB connection:

import oracledb

conn = oracledb.connect(
    user="ADMIN",
    password=admin_password,
    dsn="pythonadb_high",
    config_dir="/home/opc/wallet",
    wallet_location="/home/opc/wallet",
    wallet_password=wallet_password
)

cursor = conn.cursor()
cursor.execute("SELECT 'Connected at ' || TO_CHAR(SYSDATE, 'HH24:MI:SS') FROM dual")
print(cursor.fetchone()[0])
All scripts use the modern python-oracledb driver — no Oracle Instant Client installation required. Install with pip3 install -r requirements.txt and you are ready to go.
