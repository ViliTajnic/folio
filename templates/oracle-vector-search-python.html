005 — Oracle Vector Search in Python

Storing embeddings in Oracle 26ai, running semantic similarity search, and wrapping it in a small Flask web app — all locally

1. What We Are Building

This article builds directly on top of article 004 — you need Oracle 26ai running locally in Docker before starting here.

We will build a small semantic search system in three parts:

  • A Python script that loads a FAQ dataset, generates text embeddings using sentence-transformers, and stores them in an Oracle 26ai VECTOR column
  • A search script that takes a plain-English question, embeds it, and finds the most semantically similar answers in the database using Oracle's VECTOR_DISTANCE function
  • A minimal Flask web app that wraps the search into a local browser interface

No external API keys or cloud services required. Everything runs on your laptop against the local Oracle instance from article 004, after installing the required packages.

2. Prerequisites and Setup

Install the required Python packages:

pip install oracledb sentence-transformers flask

Make sure your Oracle 26ai container is running and your Connection Manager entry is set up:

docker start oracle26ai

3. Create the Vector Table

Connect to your local Oracle 26ai instance and create a table with a VECTOR column. We are using 384 dimensions to match the all-MiniLM-L6-v2 model we will use for embeddings:

-- Run inside SQLcl (sql -name dev26ai)
CREATE TABLE faq_items (
    id        NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    question  VARCHAR2(1000),
    answer    CLOB,
    embedding VECTOR(384, FLOAT32)
);

-- Verify the table was created
DESC faq_items;
VECTOR(384, FLOAT32) — the two parameters are the number of dimensions and the numeric type. FLOAT32 is the right balance of precision and storage for most use cases. The dimension count must match your embedding model exactly.

4. Load and Embed the FAQ Data

Create a file called load_faq.py. This script defines a small FAQ dataset, generates an embedding for each question using sentence-transformers, and inserts everything into Oracle:

from sentence_transformers import SentenceTransformer
import array

     "A vector embedding is a numerical representation of text (or other data) as a list of "
     "floating-point numbers. Semantically similar texts produce embeddings that are close "

     "HNSW (Hierarchical Navigable Small World) is an approximate nearest-neighbour index "

    ("How do I store embeddings in Oracle?",
     "Create a table with a VECTOR column, generate embeddings in Python using a model like "

# --- Load embedding model ---
print("Loading embedding model...")
model = SentenceTransformer("all-MiniLM-L6-v2")

# --- Connect to Oracle ---
# Note: using SYSTEM here for simplicity in this local demo only.
# For any real application use a dedicated least-privilege schema user.
conn = oracledb.connect(
    user="system",
    password="YourStrongPassword",
    dsn="localhost:1521/FREEPDB1"
)

# --- Clear existing data ---
cursor.execute("TRUNCATE TABLE faq_items")

# --- Insert FAQ items with embeddings ---
print(f"Embedding and inserting {len(FAQ)} FAQ items...")
for question, answer in FAQ:
    embedding = model.encode(question).tolist()
    vec = array.array("f", embedding)
    cursor.execute(
        "INSERT INTO faq_items (question, answer, embedding) VALUES (:1, :2, :3)",
        [question, answer, vec]
    )

conn.commit()
cursor.close()
conn.close()
print("Done. FAQ items loaded into Oracle.")

Run it once to populate the table:

python load_faq.py
First run note: sentence-transformers downloads the all-MiniLM-L6-v2 model (~90 MB) on first use and caches it locally. Subsequent runs are instant.

5. Run a Similarity Search from Python

Create search.py — a standalone script that takes a question as input and returns the most semantically similar FAQ entries from Oracle:

import oracledb
from sentence_transformers import SentenceTransformer
import array
import sys

model = SentenceTransformer("all-MiniLM-L6-v2")

def search(query: str, top_k: int = 3):
    embedding = model.encode(query).tolist()
    vec = array.array("f", embedding)

    # Note: SYSTEM is used for this local demo only — use a dedicated user in production.
    conn = oracledb.connect(
        user="system",
        password="YourStrongPassword",
        dsn="localhost:1521/FREEPDB1"
    )
    cursor = conn.cursor()
    cursor.execute("""
        SELECT question, answer,
               VECTOR_DISTANCE(embedding, :1, COSINE) AS distance
        FROM   faq_items
        ORDER  BY distance
        FETCH  FIRST :2 ROWS ONLY
    """, [vec, top_k])

    results = cursor.fetchall()
    cursor.close()
    conn.close()
    return results

if __name__ == "__main__":
    query = " ".join(sys.argv[1:]) or "How does vector search work?"
    print(f"\nQuery: {query}\n")
    for question, answer, distance in search(query):
        print(f"  [{distance:.4f}] {question}")
        print(f"           {answer[:120]}...")
        print()

Test it from the terminal:

python search.py how do I find similar text in a database

Oracle ranks results by cosine distance — lower is more similar. A distance of 0.0 would be an exact match. Lower values generally indicate stronger semantic similarity, but useful thresholds depend on the embedding model and dataset — do not treat any fixed number as a universal rule.

6. Build the Flask Web App

Create app.py — a minimal Flask app that exposes a search form and renders results:

from flask import Flask, request, render_template_string
from search import search

app = Flask(__name__)

TEMPLATE = """
<!DOCTYPE html>
<html>
<head>
  <title>Oracle Vector Search</title>
  <style>
    body  { font-family: sans-serif; max-width: 800px; margin: 60px auto; padding: 0 20px; }
    h1    { color: #c0392b; }
    input[type=text] { width: 100%; padding: 10px; font-size: 1rem;
                       border: 1px solid #ccc; border-radius: 4px; }
    button { margin-top: 10px; padding: 10px 24px; background: #c0392b;
             color: white; border: none; border-radius: 4px; cursor: pointer; font-size: 1rem; }
    .result  { border-left: 4px solid #c0392b; padding: 12px 16px; margin: 16px 0;
               background: #f9f9f9; border-radius: 0 4px 4px 0; }
    .question { font-weight: bold; margin-bottom: 6px; }
    .score    { font-size: 0.8rem; color: #888; margin-bottom: 8px; }
    .answer   { color: #333; line-height: 1.6; }
  </style>
</head>
<body>
  <h1>Oracle 26ai — Vector Search</h1>
  <form method="post">
    <input type="text" name="query" placeholder="Ask anything..." value="{{ query }}" autofocus />
    <button type="submit">Search</button>
  </form>

  {% if results %}
    <p>Top {{ results|length }} results for <strong>{{ query }}</strong></p>
    {% for question, answer, distance in results %}
      <div class="result">
        <div class="question">{{ question }}</div>
        <div class="score">Cosine distance: {{ "%.4f"|format(distance) }}</div>
        <div class="answer">{{ answer }}</div>
      </div>
    {% endfor %}
  {% elif query %}
    <p>No results found.</p>
  {% endif %}
</body>
</html>
"""

@app.route("/", methods=["GET", "POST"])
def index():
    results = []
    query = ""
    if request.method == "POST":
        query = request.form.get("query", "").strip()
        if query:
            results = search(query, top_k=3)
    return render_template_string(TEMPLATE, results=results, query=query)

if __name__ == "__main__":
    app.run(debug=True)

Start the app:

python app.py

Open http://localhost:5000 in your browser. Type any question — even one that does not appear verbatim in the FAQ — and Oracle will return the most semantically relevant answers.

8. How It Works Under the Hood

When you type a question into the search box, four things happen in sequence:

  1. Embed the query — sentence-transformers converts your question into a 384-dimensional float vector using the same model that encoded the FAQ data
  2. Send to Oracle — the vector is passed as a bind variable to a SQL query
  3. Oracle computes distances — VECTOR_DISTANCE(embedding, :query, COSINE) computes the cosine distance between every stored embedding and your query vector, entirely inside the database engine
  4. Return top results — ORDER BY distance FETCH FIRST 3 ROWS ONLY returns the closest matches
Why cosine distance? Cosine distance measures the angle between two vectors rather than their absolute magnitude. This makes it robust to differences in text length — a short question and a long answer can still match well if they are about the same topic. Cosine distance is a common default for text embeddings; other metrics like EUCLIDEAN or DOT may suit different use cases.

9. Extending the Project

This is a minimal but fully functional vector search system. A few natural next steps:

Add an HNSW index

The similarity query in this demo works fine on a small table — Oracle ranks all rows directly. For larger datasets, a vector index makes search fast and scalable via approximate nearest-neighbour lookup:

CREATE VECTOR INDEX faq_hnsw_idx
ON faq_items(embedding)
ORGANIZATION INMEMORY NEIGHBOR
GRAPH DISTANCE COSINE
WITH TARGET ACCURACY 95;

Search over answers too

Embed the answer text instead of (or alongside) the question for broader semantic coverage:

-- Add a second vector column
ALTER TABLE faq_items
ADD answer_embedding VECTOR(384, FLOAT32);

10. Conclusion

In under 100 lines of Python you have a working semantic search system backed by Oracle 26ai. The search finds relevant results even when the query uses completely different words than the stored FAQ — because it is matching meaning, not keywords.

This is a core retrieval building block used in many RAG pipelines: embed your knowledge base, store the vectors in a database, and retrieve the most relevant chunks at query time. Full RAG pipelines typically also involve chunking, metadata filtering, reranking, prompt assembly, and answer generation — but retrieval is where it starts. Oracle 26ai handles the vector storage and retrieval natively — no separate vector database required.

Everything built here runs against the local Oracle instance from article 004 — no external API keys or cloud services required.
