001 — Python Password Checker

Checking passwords against known data breaches using the Have I Been Pwned API — without ever sending your password over the internet

1. What This Does

This is a command-line Python script that checks one or more passwords against the Have I Been Pwned (HIBP) breach database. If a password has appeared in a known data breach, the script tells you how many times — so you know to stop using it.

What makes it worth looking at is the privacy model. The password is never transmitted anywhere. Not even a hash of it. The script uses a technique called k-anonymity to query the API without revealing what it is looking for. More on that below.

Source: github.com/ViliTajnic/passwordchecker

2. The k-Anonymity Model

The HIBP API does not accept a full password or a full hash — it accepts only the first 5 characters of a SHA-1 hash. It then returns a list of all hash suffixes in its database that start with those 5 characters, along with breach counts.

The full matching is done locally. The server never sees enough to reconstruct the original query. This is the k-anonymity guarantee: among the many hashes that share a 5-character prefix, the API cannot tell which one you are checking.

StepWhere it happensWhat is sent / kept
SHA-1 hash the passwordLocalNothing sent yet
Send first 5 hex chars of hash→ HIBP APIe.g. 5BAA6
Receive matching hash suffixes + counts← HIBP APIHundreds of partial hashes
Check if full hash suffix is in the listLocalPassword never leaves machine

3. The Complete Code

The entire script is 30 lines. Here it is in full, exactly as it appears in the repository:

import requests
import hashlib
import sys

def request_api_data(query_char):
    url = 'https://api.pwnedpasswords.com/range/' + query_char
    res = requests.get(url)
    if res.status_code != 200:
        raise RuntimeError(f'Error fetching: {res.status_code}, check the api and try again')
    return res

def get_password_leaks_count(hashes, hash_to_check):
    hashes = (line.split(':') for line in hashes.text.splitlines())
    for h, count in hashes:
        if h == hash_to_check:
            return count
    return 0

def pwned_api_check(password):
    sha1password = hashlib.sha1(password.encode('utf-8')).hexdigest().upper()
    first5_char, tail = sha1password[:5], sha1password[5:]
    response = request_api_data(first5_char)
    return get_password_leaks_count(response, tail)

def main(args):
    for password in args:
        count = pwned_api_check(password)
        if count:
            print(f'{password} was found {count} times... you should probably change your password!')
        else:
            print(f'{password} was NOT found. Please keep your password safe!')
    return 'Check completed succesfuly!'

if __name__ == '__main__':
    sys.exit(main(sys.argv[1:]))

4. Code Walkthrough

request_api_data

Takes the first 5 characters of the SHA-1 hash and hits the HIBP range endpoint. Raises a RuntimeError on any non-200 response so failures are explicit rather than silently returning empty results.

get_password_leaks_count

Parses the API response line by line. Each line is in the format HASH_SUFFIX:COUNT. The function uses a generator expression to split lazily — memory-efficient for the hundreds of lines the API returns. If the full hash suffix matches, the breach count is returned; otherwise 0.

pwned_api_check

The core of the privacy model. It SHA-1 hashes the password, converts to uppercase hex (matching the API's response format), and splits into the 5-char prefix sent to the API and the tail kept locally for comparison.

main

Iterates over all passwords passed as command-line arguments and prints a result for each. Multiple passwords can be checked in a single run.

5. Usage

Install the only dependency, then run with one or more passwords as arguments:

pip install requests

python checkmypass.py password123 correcthorsebatterystaple

Example output:

password123 was found 247116 times... you should probably change your password!
correcthorsebatterystaple was NOT found. Please keep your password safe!

6. Code Notes

The script works correctly. A few things worth noting:

  • count is a string — get_password_leaks_count returns the count as a string (from line.split(':')). The if count: check in main works because a non-empty string is truthy in Python, and it prints fine. If you wanted to compare it numerically (e.g. if count > 10), you would need int(count).
  • sys.exit pattern — sys.exit(main(sys.argv[1:])) passes the return string 'Check completed succesfuly!' to sys.exit. On most systems, a non-integer argument to sys.exit prints the string to stderr and exits with code 1. For a clean exit, calling main() directly without wrapping in sys.exit is cleaner.
  • Typo — 'Check completed succesfuly!' — "succesfuly" should be "successfully". Minor, but worth fixing if you fork the project.
  • No rate limiting or retry logic — fine for personal use; the HIBP API is generous. For batch checking of many passwords, adding a small delay between requests is courteous.

7. Why This Project

It is a small script, but it demonstrates several things clearly: working with external APIs, using hashlib for cryptographic operations, understanding a real-world privacy technique, and structuring Python code into focused single-purpose functions. The k-anonymity model is the same one used by browser password managers and enterprise security tools — understanding it from first principles is worthwhile.
